这是一个非常敏锐且务实的产品方向调整。

你的洞察非常关键：

交互门槛过高：原先的“节点式无限画布”虽然看起来酷炫，但对用户认知负荷（Cognitive Load）极高。用户并不想“填空”，他们更习惯“对话”。

场景受限：只做“职场沟通”确实把路走窄了。同样的“AI 引导 + 结构化输出”逻辑，完全可以复用于写文档、做策划、旅游规划、学习新知识等通用场景。

分栏设计的优势：左侧（Chat）负责发散与引导，右侧（Canvas）负责收敛与沉淀。这是目前 Claude Artifacts、OpenAI Canvas 等前沿产品的验证路径，非常符合人类“边聊边记”的思维习惯。

基于你的反馈，我将 PRD 进行了全面重构。现在的核心是从**“特定场景的填空游戏”转型为“通用的双屏思考工作台”**。

AI-World MVP 产品需求文档 (PRD) - v2.0 (Pivot 版)
1. MVP 核心定位与原则
产品核心：一个**“双屏思考工作台”**。

左侧 (Chat)：是你的苏格拉底式导师。它不直接给答案，而是通过多轮对话和针对性反问，帮你理清思路。

右侧 (Board)：是你的结构化笔记/交付物。随着左侧对话的深入，右侧会自动生成、更新、完善内容的结构。

Slogan：聊透了，东西就写出来了。

目标用户：需要处理复杂非结构化任务（如写方案、做决策、学新概念、应对难题）的知识工作者。他们往往“脑子很乱，不知道从何下笔”。

设计原则：

对话即生产：用户不需要手动去右侧排版，聊天的过程就是右侧内容生成的过程。

结构化沉淀：左侧是“流”（Flow），右侧是“版”（State）。左侧负责过程，右侧负责结果。

通用性 (Agnostic)：不预设场景，根据用户输入动态匹配思维框架。

2. 核心交互框架：左对齐对话 + 右对齐白板
2.1 界面布局 (The Dual-Pane Layout)
摒弃复杂的节点连线，采用经典的 Split-Screen (50/50 或 40/60) 布局。

左侧：AI 引导对话区 (The Guide)

形态：类似 ChatGPT/Claude 的标准对话流。

功能：用户输入模糊意图 -> AI 识别 -> AI 反问/引导 -> 用户回答。

右侧：实时思考白板 (The Board)

形态：一个类似 Notion/Google Doc 的文档编辑器，但支持 AI 实时“流式写入”。

功能：用户可手动记录灵感，AI 也会根据左侧对话自动提取关键信息填入。

2.2 核心工作流 (The Core Loop)
意图触发：用户在左侧输入“我想做一个小红书营销方案”或“我不知道怎么拒绝老板的需求”。

框架识别 (Dynamic Schema)：AI 隐式识别任务类型，并在右侧生成一个初始空框架（例如：背景 -> 目标 -> 策略 -> 风险）。

引导式填充 (Socratic Filling)：

AI 在左侧提问：“在开始策略之前，我们要先明确，这次营销的核心受众是谁？是大学生还是白领？”

用户在左侧回答：“主要是刚毕业的大学生。”

实时同步 (Real-time Sync)：AI 自动将“核心受众：应届/刚毕业大学生”这一信息，写入右侧白板的对应模块中。

循环迭代：随着对话继续，右侧的内容从“大纲”变为“草稿”，最后变为“完整方案”。

3. 功能详情 (Features)
模块一：智能引导对话 (The Guide - Left Pane)
能力定义：AI 不做“有问必答”的机器，而做“打破砂锅问到底”的教练。

关键逻辑：

反问机制：当用户输入过于简略时，AI 必须追问。

User: “帮我写个周报。”

AI: “没问题。为了让周报更出彩，告诉我：本周你最想强调的一个 亮点 是什么？有没有遇到什么 卡点 需要老板支持？”

多轮收敛：AI 需维护上下文，确保每个反问都是为了填充右侧的某个空白区域。

模块二：动态协作白板 (The Board - Right Pane)
这是 MVP 的核心差异点。它不是静态文本，而是用户与 AI 的共享内存。

关键特性：

混合编辑 (Hybrid Editing)：

AI 写：根据对话自动生成/修改内容。AI 修改时，对应段落出现高亮动效（类似 Ghost Typing）。

用户写：用户随时可以直接点击右侧进行修改、补充。用户的修改 AI 应当“看见”并在后续对话中考虑。

自适应模板 (Adaptive Templates)：

如果识别是【写作】：右侧是 Markdown 文档模式。

如果识别是【决策/SWOT】：右侧可以渲染为简单的表格或四象限图（MVP 可先用 Markdown 表格代替）。

如果识别是【代码】：右侧是代码编辑器。

手动“摘录” (Pin/Clip)：

用户在左侧对话中说了一段很好的话，可以点击消息旁边的“添加到白板”按钮，直接插入右侧光标处。


1. MVP 技术实现与数据结构
4.1 核心挑战：非结构化对话 -> 结构化白板更新
我们需要一个能够“旁路监听”对话并更新白板的 AI Agent。

System Prompt 逻辑 (伪代码)：

Plaintext
Role: You are a thoughtful editor and facilitator.
Task 1 (Chat): Communicate with the user in the "Chat Window". Ask clarifying questions to flesh out their vague ideas.
Task 2 (Board): Based on the chat history, maintain a structured document in the "Board Window".

Output Format (JSON):
{
  "chat_response": "你的反问或回答...",
  "board_actions": [
    {
      "action": "update_section", // 动作：更新、追加、重写
      "section_id": "strategy_part", // 对应白板的模块
      "content": "根据用户刚才的回答，更新后的策略文本..."
    },
    {
      "action": "create_structure", // 初始化模版
      "template_type": "marketing_plan"
    }
  ]
}
4.2 技术栈建议
前端：React + Tailwind CSS。

编辑器 (右侧)：推荐使用 ProseMirror 或 Tiptap。这两个库非常适合构建“协作式编辑器”，且易于通过程序代码（AI 的输出）来插入/修改文档内容。

模型：GPT-4o 或 Claude 3.5 Sonnet（推荐 Sonnet，它的长文本结构化能力和代码/文本生成能力极强，非常适合维护右侧白板的状态）。

5. 成功衡量标准 (Metrics)
北极星指标：

Board 内容留存率：用户是否在右侧白板中留下了有效内容，并进行了导出/复制操作。

过程指标：

Conversation-to-Board Ratio：平均多少轮对话能产出 100 字的白板内容？（验证 AI 的提取效率）。

User Edit Rate：用户手动修改白板的频率。（过高说明 AI 写的不好，过低说明用户参与度不够，适中最好）。

6. 舍弃清单 (Not to Do in MVP)
复杂的图表绘制：右侧白板暂时不支持画流程图、思维导图，仅支持富文本（Markdown）、列表、表格。

多人实时协作：MVP 仅限“人 + AI”协作，不支持“人 + 人”协作。

云端历史管理：MVP 可以基于 LocalStorage 做本地存储，不强制做复杂的云端账号同步系统。

7. 用户体验流程示例 (Walkthrough)
场景：用户想做一个OKR规划

Start：用户进入网页，界面左空（Chat），右空（Board）。

Input：用户在左侧输入：“下个季度我要负责提升用户留存，但我不知道怎么定 OKR。”

AI Response：

左侧：“提升留存是个好方向。为了定出可执行的 OKR，我们需要拆解一下。目前你的产品留存率大概是多少？目标提升到多少？主要手段是什么？”

右侧（自动出现）：AI 自动在白板创建了一个标题 # Q3 用户留存 OKR 规划草案，并列出了空结构：## Objective (目标), ## Key Results (关键结果), ## Action Plan (行动)。

User Reply：用户在左侧回答：“现在是 30%，想提升到 40%。主要手段是做会员体系和召回活动。”

Sync：

左侧：“明白了。跨度 10% 很有挑战性。关于‘会员体系’，你打算怎么量化它的成功？是看会员数还是看会员续费率？”

右侧（自动更新）：

Objective: 提升平台用户留存至 40%。

Key Results:

[待定] 会员体系相关指标...

[待定] 召回活动转化率...

End：几轮对话后，右侧形成了一份完整的 OKR 表格。用户点击右侧右上角的“复制”，直接贴进飞书/钉钉。
